Developing Programs for the BURT Robotic arm for use on Primates Prosthetic Limbs

This summer, I am working in the Brain Computer Interface Laboratory of Dr. Andrew Schwartz (University of Pittsburgh) developing an algorithm to be used in projects evaluating volitional strength required for 3D movement of a robotic arm under different and random levels of resistance. Currently, the Schwartz Lab has developed a brain-controlled prosthetic arm which can realistically execute 3-dimensional motion and provide the user with basic sensory feedback. Neuronal information is decoded by microelectrodes implanted in the brain in the motor and sensory cortex which through complex computational analysis produces motion of a robotic arm. One of the next frontiers for this research is to understand how the brain reacts to uncertainty. For example, the current prototype of the human controlled robotic arm is unable to titrate the force required for different types of actions. In other words, subjects are uncertain as to exactly how much force is necessary to grip an object to not only ensure that it does not fall through their grip but also that it is not crushed due to a too firm grip. Previous research has shown that humans increase accuracy by stiffening their muscles. Therefore, to address uncertainty in prosthetics, the lab strives to develop a model for how and when the brain regulates timing and anatomical location of muscle stiffening.

My part in this project is to develop programs for the Proficio Robotic Arm created by Barrett Medical for use in primate experimentation. I developed ballistic release software along various 2d paths toward a target and an accompanying python visualization. In less technical terms, a user must first apply force on the robot to overcome some unknown force threshold. After, the robot will cease to apply the opposing force; the subject must quickly stop the robot inside a target zone. I also programmed an accompanying graphics program to orient subjects in space and clearly display the target zone. The ballistic release coupled with targets create an environment where subjects must apply enough force to overcome the threshold but not excessive force so that they can quickly adapt projected movements. I am currently working on integrating the code into Dragonfly, a messaging system which the Schwartz Lab uses for communication between robots and operating machines.

Sentence Generator Webapp

I have created a grammar learning tool, which generates a random sentence and asks students questions about the generated sentence. In the tool, student can start different lessons where the student can read a description of a certain part of speech or phrase. Then, the student will proceed to the stage where the software generates a random sentence and asks the student a question pertaining to the current lesson. The tool uses computational linguistics to generate sentences that at least somewhat make sense and are "interesting" in order to keep students engaged. The tool initially read a text to develop a wordnet but was manually refined afterwards for accuracy.

As a part of Carnegie Mellon's 15-112: Fundamentals of Programming and Computer Science class, I used this sentence generator to create a website that students can use to learn grammar.

CUDA-Accelerated Lattice Boltzmann Simulator

I used the CUDA and OpenGL packages to graphically accelerate the Lattice-Boltzmann Method (LBM) Algorithms. With parallelization and efficient memory usage, considerable optimization and efficiency is possible with LBM simulation. In this simulation, constant 2-dimensional fluid flow through a medium was modeled, and at every lattice node the fluid’s curl, probabilistic velocity vectors, and pressure are calculated and available to be plotted on the screen. Work performed in collaboration with Tom Scherlis

Modeling Diffusion Rates of a Gas in an Enclosed Space

I worked to identify a relationship between the pressure of a certain gas and diffusive flux in a closed system. With Fick’s First Law of Diffusion as basis for calculations, we used a first order differential equation to discern ideal, theorized relationship between the two parameters. We designed and performed an experiment utilizing diffusion of CO2 into air, confirming the theoretical predictions.

Visualization of the Deformation of 3D Objects for Use by Virtual Prosthetic Limbs

During the summer of 2016, in the Schwartz Lab, using MatLab, HOOPS, and ultimately Unity, I worked in the Brain Computer Interface Laboratory of Dr. Andrew Schwartz (University of Pittsburgh) with Hongwei Mao to understand the forces behind human delicate object manipulation. Currently, the Schwartz Lab has developed a brain-controlled prosthetic arm which can realistically execute 3-dimensional motion and provide the user with basic sensory feedback. Neuronal information is decoded by microelectrodes implanted in the brain in the motor and sensory cortex which through complex computational analysis produces motion of a robotic arm. Although a user of the arm can instruct the arm to move in a particular direction, that user cannot specify the force required to move that specified location – the machine is instructed by a program the speed and force. One of the next frontiers for this research is to understand how the brain encodes force especially in the context of object manipulation. For example, the current prototype of the human controlled robotic arm is unable to titrate the force required for different types of actions. In other words, subjects are uncertain as to exactly how much force is necessary to grip an object to not only ensure that it does not fall through their grip but also that it is not crushed due to a too firm grip. Therefore, to address uncertainty in prosthetics, the lab strives to develop a model for how and when the brain regulates accurately forces in object manipulation. My group’s part was to develop a virtual space which a virtual finger can realistically deform an object by exerting a force on it and an accompanying learning algorithm to observe how the brain encodes specific forces and create a model for that force. The program registers the collection of forces applied to the object and using Finite Element Analysis displays the deformed object at each time step and provides inertial feedback to the finger. To develop an accurate relationship the force required to deform the object to a certain degree and firing rate of neurons, the program is complete control of the finger, and the person simply observes. However, as the person observes this movement, they are instructed to think about the same movement. During this time, the algorithm develops a baseline relationship between neuronal firing rate and force. Then, the algorithm considers 90% of the outlined finger movement and 10% according to the developed relationship. Over time, the user gains more control of the finger and eventually when the user gains 100% control a relationship can be identified. This program will be in use during the summer and fall of 2017 for testing.

Neural Network Modeling of the Stock Market

I created a deep learning neural model the stock market. The Sigmoid function was used for the activation function. I structured my network so that it had an input layer, 2 hidden layers, and a an output layer with each hidden layer containing 20 neurons. Our model worked with an average percent error of 0.7% over four week for the NASDAQ. We found that the network predicted stock indexes much better than individual stocks.
